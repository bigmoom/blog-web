表：数组和链表

对于表，我们可以普遍地认为有数组和链表两种实现形式，即能够连续存储一些数的抽象数据类型（ADT）

表的数组实现

基础知识

数组是一种非常简单的数据结构，是由相同类型的元素组成的数据集合，在内存中按顺序连续存储数据。

对于数组的数据，我们可以非常容易的通过下标获取和修改数据，其时间复杂度为O(1)O(1)O(1)，但是对于插入数据和删除数据则非常的复杂，因为插入删除数据涉及到需要将部分数据全部后移或者迁移一位，开销非常大。

此外，创建数组的时候需要指定数组的大小，并且根据容量分配内存，这就导致有的时候我们为了初始化较大容量的数组结果空间利用效率极低。为了处理空间效率不高的情况，便设计出了动态数组，会自动根据数组内部容量分配新的空间。

双指针

双指针是一种非常常见的解题方法，一般我们可以用两个相反或者相同方向的指针扫描数组以达到解题目的。

排序数组中的两个数字之和

LeetCode 167

输入一个递增排序的数组和一个值k，请问如何在数组中找出两个和为k的数字并返回他们的下标？

假设数组中存在且只存在一对符合条件的数字，同时一个数字不能使用两次

输入： [1,2,4,6,10],k=8

输出：[1,3]

  1. 最常规的解法就是遍历数组，然后对于每个数再遍历其余数字看是否有匹配的数字。很明显时间复杂度为O(N2)O(N^2)O(N2)

  2. 对于第一种方法的改良，就是在第二层遍历的时候改成二分查找（因为是有序数组）。因为二分查找的时间复杂度为O(logN)O(logN)O(logN)，所以总的时间复杂度为O(NlogN)O(NlogN)O(NlogN)

  3. 空间换时间，我们可以构造HashMap，键为数字的值，值为该数字的下标。那么我们只需要遍历一次数组，对于每个数在哈希表中查找是否存在匹配的数。时间复杂度为O(N)O(N)O(N)，空间复杂度为O(N)O(N)O(N)

  4. 因为是递增有序数组，我们可以设置两个首尾指针，当两个指针指向的数字和等于目标值时返回下标，当数字和小于目标值时我们只需要将首指针向右移动，因为右边的数字要更大一点，而数字和大于目标值时我们将尾指针向左移动即可。

数组中和为0的三个数字

LeetCode 15

输入一个数组，请问如何找出数组中所有和为0的3个数字的三元组？

返回值中不得包含重复的三元组

输入：[-1,0,1,2,-1,-4]

输出：[-1,0,1],[-1,-1,2]

求三个数字的和可以看做是上面求两个数字和的加强版，因此我们可以把问题继续转化为固定一个数字，然后对剩余两个数字求和。这样我们就只需要在上面问题上再套一层循环即可，具体操作的时间复杂度为O(N2)O(N^2)O(N2)

此外，该题中给的数据是无序的，所以我们先要对数据进行排序，排序算法的时间复杂度为O(NlogN)O(NlogN)O(NlogN)，小于O(N2)O(N^2)O(N2)，所以可以忽略不计。

题目中还要求我们不得包含重复的三元组，所以我们需要在找到匹配三元组之后，需要将指向第一个数的指针向右移动，直到数字有变化。

和大于或等于k的最短子数组

LeetCode 209

输入一个正整数组和一个正整数k，请问数组中和大于k的连续子数组的最短长度是多少？如果不存在，返回0。

输入：[5,1,4,3],k=7

输出：2

LeetCode 862 去除了正整数的条件，求最短子数组长度

我们可以用两个指针P1,P2分别指向子数组的首尾，如果子数组的和大于k，那么P1向右移动一位。因为数组为正整数组，所以右移一位即可缩小子数组的和，如果子数组的和小于k，那么P2向左移动一位。

其实，我们可以换一个角度，就是我们试图从头开始找包括左指针的且和大于等于k最小子数组,类似一个滑动窗口，找到之后便把最左端的数字删除，继续搜寻以下一个数字开始的最小子数组，只不过不用每次都遍历那么复杂。

乘积小于k的子数组

LeetCode 713

输入一个正整数组成的数组和一个正整数k，请问数组中有多少个乘积小于k的连续子数组？

输入：[10,5,2,6],k=100

输出：[10],[5],[2],[6],[10,5],[5,2],[5,2,6],[2,6]

其实这题和上面思路差不多，都还是可以用两个指针P1,P2，当子数组的乘积大于等于k时，我们右移指针P1，即剔除掉最左边的数字，然后计算出包含该数字的子数组的个数。当小于k时，我们右移指针P2直到末尾或者子数组积大于等于k。

累加数组数字求子数组之和

使用双指针的条件是数组中所有的数都是正数，但是当存在负数的时候就双指针就不适用了，因为如果有负数，就不能通过左移右移来明确使得子数组的和或者积增大减小。所以我们换一种思路来求解子数组和相关的问题。

我们先对数组进行预处理，计算从数组下标0开始到每个数字为结尾的子数组之和。我们可以记S0S_0S0​为下标0到下标0的子数组和，S1S_1S1​为下标0到下标1的子数组和，以此类推，SjS_jSj​即为下标0到下标j的子数组和，那么以下标i到下标j的子数组和就是Sj−SiS_j-S_iSj​−Si​

和为k的子数组

LeetCode 560

输入一个整数数组和一个整数k,请问有多少个数字之和等于k的连续子数组？

输入：[1,1,1],k=2

输出：2

按照常规思路，我们求导前缀和之和还是要遍历nums中的i，对于每个i，有哪些j使得子数组的和为k，这样的时间复杂度很明显是O(N2)O(N^2)O(N2)，但是其实我们可以完全不用在乎是哪个i,j得到的子数组符合条件，所以我们这里借用HashMap来存储前缀和出现的次数。

我们对目标方程进行一个转化为

prefixSum2−prefixSum1=k→prefixSum1=prefixSum2−kprefixSum2-prefixSum1=k \rightarrow prefixSum1 = prefixSum2-k prefixSum2−prefixSum1=k→prefixSum1=prefixSum2−k

所以当我们得到prefixSum2的时候，直接查看HashMap中有多少个符合条件的子数组即可，时间复杂度为O(N)O(N)O(N)

对于代码中第七行prefixSumMap.put(0,1)，因为我们这边prefixSum2-prefixSum1其实就直接忽略了从下标0开始的子数组，而是直接从下标1开始，例如[1,2,3,6],k=6，对应的前缀和为[1,3,6,12]，如果没有(0,1)，那么我么只能得到12-6=6，就忽略了6-0=6即忽略了[1,2,3]这个子数组

简单链表

基础知识

链表不同于数组，我们初始化链表的时候不需要知道链表的长度，因为链表内存分配是每添加一个节点分配一次内存，当插入一个节点时，只需要为新节点分配内存，然后通过调整指针的指向来确定该节点被链接到链表中

相比于数组，链表更适合频繁修改的数据，因为链表的插入操作只需要O(1)O(1)O(1)的时间复杂度，但是因为数据不是内存连续，链表中数据的查询效率极低，只能从头遍历，时间复杂度为O(N)O(N)O(N)

双指针

链表里面的双指针主要是两种思路：

  1. 前后双指针：即同速度的两个指针不同时间从节点出发，例如两个指针，一个指针移动k个节点后第二个指针出发，这样当第一个指针到达尾结点是，第二个指针指向的便是倒数第k个节点
  2. 快慢指针：两个指针以不同的速度或者步长从节点出发，经典用于判断链表是否有环

输入：链表 1,2,3,4,5,6 k=2

输出：链表 1,2,3,4,6

这道题其实非常简单，就用前后双指针就可以实现，这也是双指针的一个经典应用问题。要注意的就是，我们最好是使用哨兵节点来处理一些边界问题，例如k=n的情况下，就涉及到删除头节点的操作。

链表中环的入口节点

我们设置一个快慢指针，慢指针每次走一步，快指针每次走两步，那么如果没有环，快指针必然会到达null，而如果有环，怎么快慢指针必然在环内相遇。

通过上面的方法我们也可以直接得到快慢指针在环内相遇的节点位置。

那么我们接下来便是要解决如何找到环的入口

我们可以观察上面的链表，可以发现一个公式：

X+Y+K1(Z+Y)=2(X+Y+K2(Z+Y))X+Y+K_1(Z+Y) = 2(X+Y+K_2(Z+Y)) X+Y+K1​(Z+Y)=2(X+Y+K2​(Z+Y))

其中YYY是从环入口到快慢指针相遇的节点距离，K1K_1K1​为快指针在相遇前走的环的圈数，K2K_2K2​为慢指针在相遇前走的环的圈数

我们进行变换可以得到：

(K1−2K2)(Z+Y)=X+Y(K_1-2K_2)(Z+Y)=X+Y (K1​−2K2​)(Z+Y)=X+Y

因为Z+YZ+YZ+Y是整个环的长度，K1−2K2K_1-2K_2K1​−2K2​也必然是整数

所以X+YX+YX+Y也必然是整个圈长度的整数倍，即从快慢指针相遇节点再走XXX步必然会到达环的起点。

我们便可以再设置两个指针，一个指针从头节点出发，一个指针从快慢指针相遇点出发。两个指针同时同速出发，相遇点就是我们要找的环的入口节点。

两个链表的第一个重合节点

LeetCode 160

输入两个单向链表，请问如何找到他们的第1个重合节点

两个重合链表如下图所示

该题有很多种解法：

  • 暴力解法：遍历链表A，对于每个节点，遍历B,比较节点是否相同，相同的第一个节点便是重合的第一个节点
  • 构造环：我们可以将链表A的尾结点链接到B的头节点，那么就构造出一个有环的链表，问题转化为求链表中环的入口节点
  • 巧法：我们发现链表尾部都是完全重合的，所以我们可以用两个指针分别指向两个链表的头节点，在较长的链表中的指针先走，直到剩余链表的长度与较短链表一致，然后两个指针一起运动，分别比较两个指针是否相同，相同的节点即为第一个重合节点

重排链表

LeetCode 143

给定一个链表，链表中节点的顺序为

L0 → L1 → … → Ln-1 → Ln

将其重排后变成：

L0 → Ln → L1 → Ln-1 → L2 → Ln-2 → …

我们仔细观察，其实就是将链表分为两半，然后对后半部分进行反转，最后前半部分分别插入后半部分的节点

找到中点之后只需要将后半段反转，然后两个链表合并即可。

排序的循环链表

LeetCode 708

给定循环升序列表中的一个点，写一个函数向这个列表中插入一个新元素 insertVal ，使这个列表仍然是循环升序的。

给定的可以是这个列表中任意一个顶点的指针，并不一定是这个列表中最小元素的指针。

如果有多个满足条件的插入位置，可以选择任意一个位置插入新的值，插入后整个列表仍然保持有序。

如果链表为空（给定的节点是 null），需要创建一个循环有序列表并返回这个节点。否则。请返回原先给定的节点。

输入：head = [3,4,1], insertVal = 2

输出：[3,4,1,2]

这道题其实主要是注意几种特殊情况

  1. 链表为空：我们需要将该节点链向该节点自己
  2. 链表只有一个元素：两个节点互相链接
  3. 插入节点的值比链表中所有节点都大或者都小：我们将该节点放在最小和最大值之间

其他的就很简单，我们只需要找到两个连续节点，前一个节点比插入值小，后一个节点比插入值大，将值插入到这两个节点之中即可。
