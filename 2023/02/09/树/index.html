树

树的基础知识

树是一种非常常见也非常重要的数据结构，下图所示就是一个普通的树：

树的结点：

  • 结点：使用树结构存储的每一个数据元素都被称为”结点“，例如图中A就是该树的一个结点
  • 父结点（双亲结点）：上图所示，A是B,C,D的父结点，B是E,F的结点
  • 子结点：有父结点就必然有子结点，相应的B,C,D都是A的子结点
  • 兄弟结点：具有相同父结点的子结点互为兄弟结点
  • 根结点：树中没有父结点的那个结点就是根结点，可以说是树的起点
  • 叶子结点：没有任何子结点的结点，例如K,L,F,G,M,I,J都是叶子结点

子树和空树：

  • 子树：例如上图由B,E,F,K,L构成的以B结点为根结点的树就是原树的一个子树
  • 空树：没有结点的树

结点的度和层次

  • 度：对于一个结点，有多少分支称为结点的度，例如结点A的度为3，一棵树的度为树中所有结点的最大度
  • 层次：树根所在层为第一层，根的孩子结点所在层为第二层，依次类推

深度和高度

  • 深度：对于任意结点n,n的深度为从根到n的唯一路径长，根的深度为0；
  • 高度：对于任意节点n,n的高度为从n到一片树叶的最长路径长，所有树叶的高度为0；

有序树和无序树

树中每个结点的子结点谁在左边谁在右边有明确规定的称为有序树，否则为无序树，例如二叉搜索树就是有序树

森林

由\(m(m>=0)\)个互不相交的树组成的集合为森林，例如图中由B,C,D为根结点的三棵子树组成的集合便是森林。所以我们可以说一棵树是由根结点与森林组成

树的应用

二叉树的深度优先算法

二叉树是一种非常典型的具有递归性质的数据结构。因为二叉树的根结点可能具有子结点，而子结点又是对应子树的根结点，它也可能有自己的子结点。二叉树的深度优先算法便是使用二叉树的递归性质。

中序遍历

中序遍历的顺序是先遍历二叉树的左子树，再遍历二叉树的根结点，最后遍历二叉树的右子树。因此中序遍历可以直接使用递归实现

但是递归会存在局限性，即如果二叉树的深度太大时，递归的代码可能会导致调用栈溢出，所以我们这里使用栈进行调整。

节点 node 的子树为 node 本身加上所有 node 的后代。

我们分析什么样的结点是需要被删除的。首先，这个结点如果有子树，那么所有的子树的所有结点都为0，即如果一个结点需要被删除，那么他的子树的所有结点也是要被删除的。最后再检查自己的值是不是0，如果是0再删除自己。

向下的路径节点值之和

LeetCode 437

给定一个二叉树的根节点 root ，和一个整数 targetSum ，求该二叉树里节点值之和等于 targetSum 的 路径 的数目。

路径 不需要从根节点开始，也不需要在叶子节点结束，但是路径方向必须是向下的（只能从父节点到子节点）。

这道题有两种做法，第一种比较简单，就是双重递归，第二种则使用到DFS以及前缀和的知识

  • 双重递归：我们先不把问题理解为中间任意一段路径，而是依旧当做从根节点到其他子结点的路径和，只不过我们递归继续将该结点的左右结点当做根节点。DFS求路径是一层递归，把子结点当做父亲结点求解问题又是一层递归，所以是双层递归

  • 前缀和：前缀和我们在数组那块使用过，这里求连续子路径的和与之前求连续子数组的和类似，依旧可以使用前缀和来进行求解。即得到公式： \[ prefixSum[b]-prefixSum[a]=target \] 我们使用DFS计算prefixSum并把值存在Hashmap中，Hashmap中Key就是子路径的长,value就是该长度子路径的个数

    我们需要注意的就是25行，我们在递归结束之后需要把当前节点在前缀和中删除

二叉搜索树（BST）

二叉搜索树即具有对于每个结点X，它的左子树中的所有项都小于X中的项，右子树的所有项都大于X中的项的二叉树。因此二叉查找树的平均深度为\(O(logN)\)

其中我们需要注意的就是remove方法，其中如果删除的时候同时具有左右结点，那么我们需要将该结点用右子树中最小的结点进行替换。

二叉树的三种不同的深度优先遍历算法都非常适用于二叉搜索树，其中中序遍历最为常用，因为二叉搜索树的中序遍历可以得到按结点值递增的序列

二叉搜索树的下一个结点

LeetCode 285

给定一棵二叉搜索树和其中的一个节点 p ，找到该节点在树中的中序后继。如果节点没有中序后继，请返回 null 。

节点 p 的后继是值比 p.val 大的节点中键值最小的节点。

  • \(O(n)\)解法

    直接对整个树进行中序遍历，并且设置一个布尔值，判断是否碰到输入的结点p，在碰到结点p之后第一个遇到的结点就是输出结点

  • \(O(h)\)解法，\(h\)是树的高度

    我们其实可以发现，我们要找的就是大于p的最小的结点。我们可以用一个变量存储这个结点，然后根据结点大小进行逐层比较。其实，这个结点要么就是结点p的子结点构成子树的最左端结点或者当结点p没有右结点时，就是上一个比p结点大的结点。

所有大于或者等于结点的值之和

LeetCode 538

给出二叉 搜索 树的根节点，该树的节点值各不相同，请你将其转换为累加树（Greater Sum Tree），使每个节点 node 的新值等于原树中大于或等于 node.val 的值之和。

提醒一下，二叉搜索树满足下列约束条件：

  • 节点的左子树仅包含键 小于 节点键的节点。
  • 节点的右子树仅包含键 大于 节点键的节点。
  • 左右子树也必须是二叉搜索树。

因为涉及到大小的顺序，所以我们肯定还是选择中序遍历。当我们中序遍历的时候，其实就已经知道小于这个结点的所有结点，那么我们将树的值累和减去前置的累和即大于等于该结点值的累和。但是这样我们就需要多一次遍历，去计算所有结点的累和。

其实我们可以更改中序遍历的顺序，即右根左，那么我们遍历的时候就知道了所有大于自己的结点，即一次遍历即可。

二叉搜索树迭代器

LeetCode 173

实现一个二叉搜索树迭代器类BSTIterator ，表示一个按中序遍历二叉搜索树（BST）的迭代器：

  • BSTIterator(TreeNode root) 初始化 BSTIterator 类的一个对象。BST 的根节点 root 会作为构造函数的一部分给出。指针应初始化为一个不存在于 BST 中的数字，且该数字小于 BST 中的任何元素。
  • boolean hasNext() 如果向指针右侧遍历存在数字，则返回 true ；否则返回 false 。
  • int next()将指针向右移动，然后返回指针处的数字。

注意，指针初始化为一个不存在于 BST 中的数字，所以对 next() 的首次调用将返回 BST 中的最小元素。

你可以假设 next() 调用总是有效的，也就是说，当调用 next() 时，BST 的中序遍历中至少存在一个下一个数字。

二叉搜索树的迭代器其实也还是中序遍历，我们一种方法是可以使用一个存储所有结点，中序遍历这个树，依次往链表里面添加值，这样我们其实也就是将二叉树展平，二叉搜索树的迭代器变成了链表的一个迭代器，next(),hasNext()的时间复杂度很明显是\(O(1)\)，因为使用到一个存放\(n\)个结点的数据结构，所以空间复杂度为\(O(n)\)

其实我们再看中序遍历，其实就是将迭代器的不同部分组合在了一起。最外层的while循环即hasNext()，里面的while循环即next()，这样我们需要使用的就只有中序遍历时候用到的栈，空间复杂度为\(O(h)\)，对于时间复杂度，如果二叉搜索树有\(n\)个结点，那么调用\(n\)次next才能遍历所有结点，所以均摊时间复杂度为\(O(1)\)

二叉搜索树中的两个结点的值之和

给定一个二叉搜索树 root 和一个目标结果 k，如果二叉搜索树中存在两个元素且它们的和等于给定的目标结果，则返回 true。

很直观的想法就是将每个结点的值都存在一个HashMap中，然后遍历所有结点，直接到Hashmap找有没有符合目标的结点，如果有直接返回。很明显，时间复杂度为\(O(N)\)，空间复杂度也为\(O(N)\)

但是这样的话，我们就没有利用到二叉搜索树的特征了，就是对于所有二叉树都可以做。对于二叉搜索树，我们可以进一步的优化。对于二叉搜索树的中序遍历，我们知道是一个递增序列，那么题目就相当于转换成了在递增序列里面找到是否存在和为特定值的两个数。这其实就是数组章节我们做过的使用两个指针模拟滑动窗口来求解问题。所以我们这里也可以使用两个指针。

为了节省空间开销，我们并不是直接转化为数组的滑动窗口，而是直接构造两个类似于上一题的中序搜索迭代器，只不过一个是中序一个是逆中序的遍历方式

自平衡二叉搜索树

对于理想的二叉搜索树，查找的时间复杂度为\(O(logN)\)，但是由于没有对二叉搜索树进行限制，很有可能出现倾斜严重的情况，最坏的情况即退化为一条链表，查找时间复杂度为\(O(N)\)，这是我们不想见到的，所以我们构建出自平衡二叉搜索树。

AVL

AVL，严格平衡的二叉搜索树，它能保证树中任意结点的两个子树高度差不大于1。

如上图所示，左边的树即为AVL，右边则不是，因为结点7的左右结点高度相差为2>1

[AVL构造可视化体验]](https://www.cs.usfca.edu/~galles/visualization/AVLtree.html)

由于插入和删除操作，可能会导致不平衡的情况，对于不平衡的状况可以分为四种LL,LR,RR,RL

  • LL：插入或删除一个节点后，根节点的左子树的左子树还有非空子节点，导致"根的左子树的高度"比"根的右子树的高度"大2，导致AVL树失去了平衡。
  • LR：插入或删除一个节点后，根节点的左子树的右子树还有非空子节点，导致"根的左子树的高度"比"根的右子树的高度"大2，导致AVL树失去了平衡。
  • RR：插入或删除一个节点后，根节点的右子树的右子树还有非空子节点，导致"根的右子树的高度"比"根的左子树的高度"大2，导致AVL树失去了平衡。
  • RL：插入或删除一个节点后，根节点的右子树的左子树还有非空子节点，导致"根的右子树的高度"比"根的左子树的高度"大2，导致AVL树失去了平衡。

那么对于这些失衡状态我们需要进行旋转操作以恢复平衡

具体体验可以到上面的可视化链接仔细感受

  • 左旋：

    （1）节点的右孩子替代此节点位置 （2）右孩子的左子树变为该节点的右子树 （3）节点本身变为右孩子的左子树

  • 右旋：

    （1）节点的左孩子代表此节点 （2）节点的左孩子的右子树变为节点的左子树 （3）将此节点作为左孩子节点的右子树。

失衡状态 旋转方式
LL 右旋
LR 左旋再右旋
RL 右旋再左旋
RR 左旋

其实LR,RL都可以理解为先将失衡状态转化为LL,RR然后再处理

红黑树（RBT）

红黑树是一种含有红黑结点并能自平衡的二叉查找树。它必须满足下面性质：

  • 所有结点只能是红色或者黑色
  • 根结点必须是黑色
  • NULL结点是黑色
  • 红色结点的父节点以及孩子结点必须是黑色
  • 所有结点到NULL结点的路径中必须有相同数量的黑色结点

首先我们需要知道为什么我们需要使用红黑树

红黑树也是一种平衡二叉搜索树，但是与AVL不同的是，红黑树舍弃了绝对的平衡（但也不是不平衡，他能保证树的高度最多为\(2log(N+1)\)，所以查找操作仍然是对数复杂度），换来的时候简化了恢复平衡的操作。红黑树通过给颜色上色，从而可以通过重新着色的操作代替旋转操作，操作更快并且可以有效降低为了恢复平衡的旋转操作。红黑树可以在三次旋转内恢复树的平衡。

红黑树的插入操作和删除操作较AVL还是要复杂很多的

    • Uncle结点为红色

      交换grandpa结点与parent,uncle结点的颜色，保证所有结点到NULL结点的路径中必须有相同数量的黑色结点

      但是因为grandpa结点变红，我们需要继续向上检查是否平衡

    • 没有uncle结点或者uncle结点为黑色

      • 插入结点cur为父结点的左子结点且parent结点为grandpa结点的左结点，即LL

        跟AVL中类似，右旋操作然后将parent颜色改为黑色，grandpa结点变为红色

        RR情况对称，不再赘述

      • 插入结点cur为父亲结点的右子结点，parent结点为grandpa结点的左结点，即LR

        跟AVL中类似，先对parent结点左旋，然后就回到了刚刚上面LL的情况，处理方式一样

        RL情况对称，不再赘述

删除操作

删除操作比插入操作要更复杂一些

首先我们先将即将被删除的结点分为三种：

  1. 叶子结点
  2. 只有一个孩子结点
  3. 有两个孩子结点

其实我们知道，在二叉搜索树中删除具有两个孩子结点的结点都是使用中序后继结点（右子树中最小的那个结点）来替代删除结点，所以第三种情况可以转化为删除待删除结点的右子树中最小的那个结点，而最小结点必然要么是叶子结点，要么是只有右子树。所以删除结点的情况其实只有1,2两种情况

接下来我们继续对1,2两种情况细分

  1. 删除结点为红色结点，我们有上面两种情况知道，如果这个结点是红色结点，那么她必然是叶子结点，因为红色结点不可能只有一个黑色结点。

    对于这种情况，直接删除即可

  2. 删除结点为黑色结点

    1. 该结点只有一个孩子结点

      如上图所示，我们只需要用结点的孩子结点替换删除结点，并且将颜色改成黑色即可

    2. 删除结点为黑色叶子结点

      1. 兄弟结点为黑色结点且远侄子结点为红色结点

        没有上色的结点表示任意颜色均可

        我们可以发现，如果删除D，那么左边的黑色结点个数必然会比右边少一，那么为了平衡，我们将右边结点借过来即可

        对结点P左旋，并变色为黑色，然后将SR结点变色为黑色，即可以保证左右子树黑色结点数相同

        D为右孩子，远侄子结点为SL情况对称，不再赘述

      2. 兄弟结点为黑色结点且近侄子结点为红色

        与上面情况类似，我们可以将近侄子变成远侄子，即右旋结点S

        转化为第一种情况，相同处理

      3. 兄弟结点为红色结点

        我们操作是将结点P左旋，然后更换P和S的颜色

        这个时候我们就可以发现情况变成了兄弟结点为黑色结点的情况，相同方式处理

      4. 兄弟结点黑色叶子结点

        1. 父亲结点为红色结点

          很明显，我们只需要将P结点和S结点的颜色互换即可

        2. 父亲结点为黑色结点

          因为删除D结点后，左右子树黑色结点个数不同，所以将S结点变成红色

          但是这样从P开始的路径黑色结点个数就比之前要少一，所以我们需要继续向上维护，我们将P结点作为删除结点继续维护平衡，不过不删除P结点

红黑树凭借着高效的查找效率以及较低的维护开销在许多方面都有使用，例如Hashmap设置链表长度阈值为8，达到阈值就将链表转换为红黑树。还有TreeMap和TreeSet都是直接使用红黑树实现

这里阈值为8是因为经过统计计算，冲突次数为8的概率非常非常低，几乎为0，但是这样的概率还是发生了说明可能由于hash函数或者数据的原因，冲突会常常发生，这时候转化为红黑树。

红黑树转链表的阈值为6，如果将这个值也设置为8，那么在发生碰撞的时候可能会来回转换，浪费资源

TreeSet和TreeMap的应用

TreeSet和TreeMap就是使用二叉平衡树实现的set和map类型，与HashMap相比，虽然查询效率较低，但是提供了可以按照大小顺序进行搜索的功能，相较于排序的动态数组，虽然动态数组的查询速度可以通过排序算法达到\(O(logN)\)，但是对于动态数组的插入和删除操作还是\(O(N)\)

TreeSet常用函数

序号 函数 函数功能
1 ceiling 返回键大于或等于给定值的最小键，如果没有则返回null
2 floor 返回键小于或等于给定值的最大键，如果没有则返回null
3 higher 返回键大于给定值的最小键，如果没有则返回null
4 lower 返回键小于给定值的最大键，如果没有则返回null

TreeMap常用函数

序号 函数 函数功能
1 ceilingEntry/ceilingKey 返回键大于或等于给定值的最小映射/键，如果没有则返回null
2 floorEntry/floorKey 返回键小于或等于给定值的最大映射/键，如果没有则返回null
3 higherEntry/higerKey 返回键大于给定值的最小映射/键，如果没有则返回null
4 lowerEntry/lowerKey 返回键小于给定值的最大映射/键，如果没有则返回null
值和下标之差都在给定的范围内

LeetCode 220

给你一个整数数组 nums 和两个整数 k 和 t 。请你判断是否存在 两个不同下标 i 和 j，使得 abs(nums[i] - nums[j]) <= t ，同时又满足 abs(i - j) <= k 。

如果存在则返回 true，不存在返回 false。

  • 滑动窗口+TreeSet

    我们可以遍历原数组，寻找在前k个数中大于自己的最小值和小于自己的最大值，如果差值绝对值小于t即满足，返回true。这里我们要找到这种大于自己的最小值和小于自己的最大值，使用TreeSet非常合适。我们遍历一个即在TreeSet中加入一个，但是最多只保留k个元素，这样便可以保证下标差值小于等于k

  • 桶排序

    我们也可以借助桶来实现要求，即我们设置大小为t+1的多个连续桶，落在同一个桶中的元素差值必然小于等于t，如果桶中没有元素，那么检测邻接桶中有没有符合的元素

    这里我们计算桶id的时候判断是否为负数，如果为负数则加一并左移一个桶的序号，例如当t=3时，-1,-2,-3,-4应该落在序号为-1的桶中。

日程表

实现一个 MyCalendar 类来存放你的日程安排。如果要添加的日程安排不会造成 重复预订 ，则可以存储这个新的日程安排。

当两个日程安排有一些时间上的交叉时（例如两个日程安排都在同一时间内），就会产生 重复预订 。

日程可以用一对整数 start 和 end 表示，这里的时间是半开区间，即 [start, end), 实数 x 的范围为， start <= x < end 。

实现 MyCalendar 类：

  • MyCalendar() 初始化日历对象。
  • boolean book(int start, int end) 如果可以将日程安排成功添加到日历中而不会导致重复预订，返回 true 。否则，返回 false 并且不要将该日程安排添加到日历中。

我们这题重点是每次加入新时间的时候，需要比较前一个事件和后一个事件，那么TreeMap就非常适合这一题
