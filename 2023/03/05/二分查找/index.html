二分查找

二分查找的基础知识

我们都知道在一个数组中搜索都是逐一搜索，时间复杂度很明显是O(N)O(N)O(N)

但是，如果一个数组是排好序的，那么我们就可以采用二分查找的方式进行优化

  • 取出数组最中间的元素进行比较，如果与目标数字相等就找到了
  • 如果中间元素比目标元素小，说明目标元素在后半段数组，对后半段数组再进行二分查找
  • 同理，如果中间元素比目标元素大，对前半段数组再进行二分查找
  • 重复二分查找直到找到目标元素

很显然，二分查找的时间复杂度为O(logN)O(logN)O(logN)

排序数组中的二分查找

查找插入位置

LeetCode 35

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

请必须使用时间复杂度为 O(log n) 的算法。

这道题要求时间复杂度为O(logN)O(logN)O(logN)的方法，就是要我们使用二分查找

我们还是普通的二分查找一样，但是就是多了一种情况，即目标不在数组中，我们要将该数插入到合适的地方，其实就只有两种情况，就是当mid大于目标值的时候，前一个数小于目标值，或者mid小于目标值的时候，后一个数大于目标值，我们其实不需要两种情况都讨论，只需要讨论一种即可

山峰数组的顶部

LeetCode 852

符合下列属性的数组 arr 称为 山脉数组 ：

  • arr.length >= 3
  • 存在 i（0 < i < arr.length - 1）使得
    • arr[0] < arr[1] < ... arr[i-1] < arr[i]
    • arr[i] > arr[i+1] > ... > arr[arr.length - 1]

给你由整数组成的山脉数组 arr ，返回任何满足 arr[0] < arr[1] < ... arr[i - 1] < arr[i] > arr[i + 1] > ... > arr[arr.length - 1] 的下标 i 。

这道题虽然并不是单调有序数组，但是其实也是有序数组，我们仍然可以使用二分查找

如果mid比左右两边都大说明找到目标数字了，如果mid比右边小，说明峰顶再右边，如果比左边小说明峰顶在左边

排序数组中只出现一次的数字

LeetCode 540

给你一个仅由整数组成的有序数组，其中每个元素都会出现两次，唯有一个数只会出现一次。

请你找出并返回只出现一次的那个数。

你设计的解决方案必须满足 O(log n) 时间复杂度和 O(1) 空间复杂度。

二分查找最重要的就是找到mid之后下一次迭代是在前半段还是后半段去搜寻以及什么时候终止迭代

这道题我们可以发现找到mid之后可以通过判断前后是否与mid相等以及奇偶性来判断这个唯一数出现在哪里

例如1122344，第一次迭代mid是2，我们发现mid-1==mid并且包括mid，前半部分是偶数个，所以唯一数必然出现在后半部分，找到规律之后二分查找就非常好写了

按权重生成随机数

LeetCode 528

给你一个 下标从 0 开始 的正整数数组 w ，其中 w[i] 代表第 i 个下标的权重。

请你实现一个函数 pickIndex ，它可以 随机地 从范围 [0, w.length - 1] 内（含 0 和 w.length - 1）选出并返回一个下标。选取下标 i 的 概率 为 w[i] / sum(w) 。

  • 例如，对于 w = [1, 3]，挑选下标 0 的概率为 1 / (1 + 3) = 0.25 （即，25%），而选取下标 1 的概率为 3 / (1 + 3) = 0.75（即，75%）。

这道题主要的就是一个权重和概率问题，我们可以把权重数组转化为一个前缀和数组

例如[1,2,3,4]的权重数组，他的前缀和数组为[1,3,6,10]，可以发现，选择下标i的概率就是前缀和数组中i,i-1两项的差值除以总的前缀和

那么我们要做的就是取随机数，看他落在那个区域即可，并且发现前缀和数组必然是递增的，因为每个权重都是正数，所以我们还是可以使用二分查找进行搜索

在数值范围内二分查找

数值范围的二分查找指的是我们一开始不知道具体的数值解，但是我们知道解的范围即最小值和最大值，那么我们使用二分查找进行探索，直到满足问题的解

求平方根

LeetCode 69

给你一个非负整数 x ，计算并返回 x 的 算术平方根 。

由于返回类型是整数，结果只保留 整数部分 ，小数部分将被 舍去 。

**注意：**不允许使用任何内置指数函数和算符，例如 pow(x, 0.5) 或者 x ** 0.5 。

这道题求解一个数的平方根，我们知道一个数x的平方根肯定在1~x范围内，平方根越大，值越大，所以我们可以使用二分查找搜索解

上面mid<=x/mid和mid+1>x/(mid+1)是为了防止溢出

狒狒吃香蕉

LeetCode 875

​ 珂珂喜欢吃香蕉。这里有 n 堆香蕉，第 i 堆中有 piles[i] 根香蕉。警卫已经离开了，将在 h 小时后回来。

珂珂可以决定她吃香蕉的速度 k （单位：根/小时）。每个小时，她将会选择一堆香蕉，从中吃掉 k 根。如果这堆香蕉少于 k 根，她将吃掉这堆的所有香蕉，然后这一小时内不会再吃更多的香蕉。

珂珂喜欢慢慢吃，但仍然想在警卫回来前吃掉所有的香蕉。

返回她可以在 h 小时内吃掉所有香蕉的最小速度 k（k 为整数）。

这道题我们也是知道一个解的范围求解的具体值，所以我们还是可以使用二分查找

吃香蕉的速度必然是大于等于1小于等于最大堆中的数量，所以我们二分查找速度，然后比较这个速度吃完香蕉需要的时间和h即可
