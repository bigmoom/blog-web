字符串

字符串的基本知识

字符串是由任意长度（长度可能为0）的字符组成，是编程语言中表示文本的数据类型，JAVA中使用String来表示字符串，与其他原因有不小的区别

String的不可变性

JAVA中String设计为不可变类

上面代码其实就是String存储字符的变量，可以发现是一个final不可变属性，这样设计的原因有三点

  • 字符串常量池的需要：字符串常量池是 Java 堆内存中⼀个特殊的存储区域, 当创建⼀个 String 对象时，假如 此字符串值已经存在于常量池中，则不会创建⼀个新的对象，⽽是引⽤已经存在的对象；
  • 允许 String 对象缓存 HashCode：Java 中 String 对象的哈希码被频繁地使⽤, ⽐如在 HashMap 等容器中。 字符串不变性保证了 hash 码的唯⼀性，因此可以放⼼地进⾏缓存。这也是⼀种性能优化⼿段，意味着不必每次都 去计算新的哈希码；
  • String 被许多的 Java 类(库)⽤来当做参数，例如：⽹络连接地址 URL、⽂件路径 path、还有反射机制所需要 的 String 参数等, 假若 String 不是固定不变的，将会引起各种安全隐患。

上面提到了字符串常量池这个概念，过会还会涉及到JVM中的栈，堆，这些都会在JVM中具体介绍

我们可以通过下面的用例来理解一下字符串常量池和字符串变量

关于String str1="abc"，这里创建了一个字符串常量，JVM首先会检查字符串常量池中是否存在该字符串对象，如果已经存在就不会再创建，直接返回字符串在字符串变量池中的内存地址，如果字符串变量池中还不存在则会在字符串变量池中创建该字符串对象，然后再返回地址。所以在String str2="abc"的时候其实就是直接返回了字符串变量池中abc的地址，所以str1==str2

但是在执行String str3 = new String("abc")的时候，JVM会首先检查字符串变量池中是否已经有该字符串对象，如果没有先创建该字符串对象，然后再到堆内存里面创建一份字符串对象，再把字符串常量池中的abc字符串内容拷贝到堆内存中字符串对象中，最后返回堆内存中该字符串的内存地址，即栈内存中存储的地址是堆内存中对象的地址。

而String str4 = new String("abc")是在堆内存中又创建了一个对象

String, StringBuilder, StringBuffer

由于String对象的不可变，所以每一个操作都会产生新的对象，这样对内存非常不友好，而且会频繁的创建对象

所以StringBuilder就是用来弥补String的不可变性，StringBuilder内部维护了一个非Final的char[] value，所以可以做到在不开辟新内存的前提先修改原字符串

StringBuffer其实功能与StringBuilder一致，只是对于字符串的修改增加了synchronized关键字，所以对于字符串的操作是线程安全的，但是也因此性能略次于StringBuffer

双指针

类似于数组的双指针，我们在字符串数组中也可以使用双指针来进行类似于字符的计数问题

字符串中的变位词

LeetCode 567

给你两个字符串 s1 和 s2 ，写一个函数来判断 s2 是否包含 s1 的排列。如果是，返回 true ；否则，返回 false 。

换句话说，s1 的排列之一是 s2 的 子串 。

这道题我们第一个想法就是枚举出s1的所有排列，然后依次到s2中找存不存在，但是这样的时间复杂度太爆炸了，所以我们换一种思路

我们发现字符串的不同排列保持不变的就是每个字符的个数，所以我们可以用一个26长度的散列表记录每个字符的数量

我们先记录s1中字符的对应个数，例如上例，这个时候a,b的个数都是1

然后我们用一个长度为s1长度的双指针扫描s2，每扫到一个新的字符，该字符-1，每有一个字符离开双指针区间+1，这样当我们搜索到变位词的时候，所有字符数量应该都是0

字符串中的所有变位词

LeetCode 438

给定两个字符串 s 和 p，找到 s 中所有 p 的 异位词 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

异位词 指由相同字母重排列形成的字符串（包括相同的字符串）。

这道题跟上面的题其实几乎一模一样，唯一不同的就是需要遍历完，并且保存每次符合异位词的下标

不含重复字符串的最长子字符串

LeetCode 3

给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。

这道题我们还是可以使用双指针来进行计数

用一个散列表记录左右指针中间的字符的个数，一开始右指针向右移动，当有字符个数大于1的时候，左指针向右移动直到所有的字符个数都小于等于1，这样以滑动窗口的模式就可以找到最长子串

包含所有字符的最短字符串

LeetCode 76（类似）

给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果 s 中不存在涵盖 t 所有字符的子串，则返回空字符串 "" 。

注意：

  • 对于 t 中重复字符，我们寻找的子字符串中该字符数量必须不少于 t 中该字符数量。
  • 如果 s 中存在这样的子串，我们保证它是唯一的答案。

这道题我们还是使用双指针

使用一个技术标志来检查左右指针区间是不是包括了t中所有字符，如果包括了就向右移动左指针，如果不包括就向右移动右指针

回文字符串

回文字符串是指一个字符串从头读和从尾读都是一样的，例如noon和madam，不过也可能是一长段的字符串，例如Sir, i demand, I am a maid named Iris， 不过这种就是不考虑非字符的回文字符串

有效的回文

LeetCode 125

如果在将所有大写字符转换为小写字符、并移除所有非字母数字字符之后，短语正着读和反着读都一样。则可以认为该短语是一个 回文串 。

字母和数字都属于字母数字字符。

给你一个字符串 s，如果它是 回文串 ，返回 true ；否则，返回 false 。

这个就非常简单，用两个指针分别指向头和尾，相向而行，一一比较，如果全部相等则是回文数

最多删除一个字符得到回文

LeetCode 680

给你一个字符串 s，最多 可以从中删除一个字符。

请你判断 s 是否能成为回文字符串：如果能，返回 true ；否则，返回 false 。

这道题和上一题类似，我们可以还是两个指针前后相向而行，遇到第一个不相等的字符，确认删除前面那个字符或者后面字符之后剩下的还是不是回文就可以了

回文子字符串的个数

LeetCode 647

给你一个字符串 s ，请你统计并返回这个字符串中 回文子串 的数目。

回文字符串 是正着读和倒过来读一样的字符串。

子字符串 是字符串中的由连续字符组成的一个序列。

具有不同开始位置或结束位置的子串，即使是由相同的字符组成，也会被视作不同的子串。

判断回文数不仅可以从两边向里也可以从中间向两边，这道题我们就可以遍历字符串s的每一个字符，然后计算以这个字符为中心或者以这个字符和下一个字符组合的为中心构造的回文子串的数量即可，例如aba就是以b为中心的回文字符串，abccba就是以cc为中心的回文字符串
