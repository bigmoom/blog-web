排序

排序是非常常见也是非常重要的一个考点，我们需要掌握不同的排序算法，以及这些排序算法之间的优劣之处

排序的评价维度

排序算法的评价维度主要有四类：稳定性，就地性，自适应性，比较类

稳定性

排序算法的稳定性其实就是只在排序之后，相同值元素在数据文件中的相对位置是否发生变化

  • 稳定排序：在完成排序后，不改变 相等元素在数组中的相对顺序
  • 非稳定排序：在完成排序后，相等元素在数组中的相对位置 可能被改变

例如对下面的数据进行排序

就地性

就地性其实就是看排序算法是否需要借助额外的空间进行排序

  • 原地排序：不需要使用额外的辅助空间
  • 非原地排序：需要使用额外的辅助空间

原地排序不使用额外空间，可以节约内存；并且一般情况下，由于数据操作减少，原地排序的运行效率也更高。

自适应性

自适应性其实就是排序算法的时间复杂度受输入数据影响

  • 自适应排序：时间复杂度受输入数据影响，即最佳 / 最差 / 平均时间复杂度不相等。
  • 非自适应排序：时间复杂度恒定，与输入数据无关。

我们对于排序算法，希望的是最差时间复杂度等于平均时间复杂度，即不希望排序算法的时间复杂度在某些输入的情况下效率劣化

比较类

比较类其实就是排序算法通过元素之间的比较算子决定元素的顺序

  • 比较类：排序基于比较算子（小于，等于，大于）
  • 非比较类：不基于比较算子进行排序

冒泡排序

算法介绍

我们以递增顺序为例，算法思路就是从左到右遍历元素，依次比较该元素和后面的元素的大小，如果前面元素大于后面元素那么交换两个元素的位置，这样一轮下来最大的元素必然已经到了最后面，那么我们继续对前面n-1个元素进行排序

  1. 对长度为n的数组nums进行排序
  2. 从左到右遍历数组，比较i和i+1大小，如果nums[i]>nums[i+1]交换两个元素
  3. 一轮结束之后，待排序数组中最大元素已经被移到最后，重复对前面未排序数组进行冒泡排序

上面我们其实是对原始的冒泡排序进行了优化，我们设置了一个早停变量，当冒泡排序的时候，如果一轮下来没有发生任何交换说明此时数据就已经是排好序了的，不需要再进行下去了

算法性能分析

  • 时间复杂度：\(O(N^2)\)
  • 空间复杂度：\(O(1)\)，不需要使用额外辅助空间，即原地排序
  • 稳定排序，不会改变数据的相对位置
  • 自适应：引入优化之后最优时间复杂度为\(O(N)\)，即输入就是排好序的，但是平均和最差时间复杂度还是\(O(N^2)\)

选择排序

算法介绍

选择排序其实和冒泡排序大方向的思路都是一样的，就是通过一次遍历得到一个最小或最大值，这样就能确定一个元素的位置

  1. 对长度为n的数组nums进行排序，这里以递增排序为例
  2. 遍历数组，找出最小值，将最小值与头部元素交换位置
  3. 重复上一步，直到数组有序

算法性能分析

  • 时间复杂度：\(O(N^2)\)

  • 空间复杂度：\(O(1)\)，不需要使用额外辅助空间，即原地排序

  • 不稳定排序，可能会改变数据中元素的相对位置，例如下图

  • 非自适应排序，没有早停机制，就算原始数据已经排好序也还是需要\(O(N^2)\)的时间复杂度

插入排序

算法介绍

插入排序是一种基于数组插入的排序，主要思路是遍历数组，遇到的新元素插入到前面已经排好序的数组中合理的位置

  1. 还是以递增排序为例，对长度为n的数组nums排序
  2. 第一轮先选取数组第二个元素，将他放到只有第一个元素的数组中的合适位置，这时前两个元素组成的子数组有序
  3. 第二轮选取第三个元素，将该元素插入到前面有序数组中，保持有序性
  4. 重复直到最后一个元素完成操作，此时整个数组有序

算法性能分析

  • 时间复杂度：\(O(N^2)\)
  • 空间复杂度：\(O(1)\)，不需要使用额外辅助空间，即原地排序
  • 稳定排序，不会改变数据的相对位置
  • 自适应：最佳情况下时间复杂度为\(O(N)\)，即每次遇到的新元素都是最大的元素

可以看到，冒泡排序，选择排序，插入排序时间复杂度都是\(O(N^2)\)，但是冒泡排序和选择排序都需要创建一个临时变量，共需三个单元操作，而插入排序只需要赋值这一个单元操作，所以插入排序较冒牌排序和选择排序更优

我们过会接下来会看到时间复杂度为\(O(NlogN)\)的排序算法，但是并非插入排序就不会使用，实际上在JAVA中对于排序算法的调用分两种情况

  • 长数组：采用时间复杂度为\(O(NlogN)\)的排序算法，例如快速排序
  • 短数组：采用插入排序，因为在数组较短的时候，每次迭代的单元操作数量占时间复杂度的主导，这个时候插入排序更快

希尔排序

算法介绍

希尔排序可以说是第一个脱离\(O(N^2)\)时间复杂度的排序算法，是插入排序的一种优化

希尔排序的主要思想就是先将数组处理成大部分有序，然后最后一步再插入排序一样，这样就能大大提升排序的效率

具体操作就是按照一定的gap值，即步长，不断的对数组进行插入排序，直到gap值为1

我们以数组[61, 109, 149, 111, 34, 2, 24, 119, 122, 27]为例

  • 我们选用经典希尔排序的gap值：N/2，N/4...，所以gap初始值为size/2=5

  • 步长为5的子数组为[61,2],[109,24],[149,119],[111,122],[34,27]

  • 对子数组进行插入排序，排序之后子数组为[2,61],[24,109],[119,149],[111,122],[27,34]

    排序之后的数组为[2,24,119,111,27,61,109,149,122,34]

  • 更新gap=gap/2=2，继续将数组分为子数组

    [2,119,27,109,122],[24,111,61,109,122]

    对子数组插入排序之后

    [2, 27, 109, 119, 122],[24, 34, 61, 111, 149]

    原数组为 [2, 24, 27, 34, 109, 61, 119, 111, 122, 149]

  • 更新gap=gap/2=1，说明这是最后一轮，直接继续插入排序即可

算法性能分析

  • 时间复杂度：希尔排序的时间复杂度比较难证明，希尔增量的最坏时间复杂度还是\(O(N^2)\),一些经过优化的增量序列如Hibbard序列\(gap = 2^k-1\)经过复杂证明可使得最坏时间复杂度为\(O(N^{3/2})\)
  • 空间复杂度：\(O(1)\)，不需要使用额外辅助空间，即原地排序
  • 非稳定排序，虽然插入排序是稳定排序，但是因为涉及到分组重组，所以会有相对位置的改变
  • 自适应：最佳情况下时间复杂度为\(O(N)\)，不同增量序列的最坏时间复杂度也不同

归并排序

算法介绍

归并排序是一种分治的思想，具有划分和合并两个阶段

  • 划分：划分是指将数组不断从中心点分为左右两端，将长数组问题划分为左右两个子数组排序问题
  • 合并：划分到数组长度为1的时候，向上合并，不断将左右两个子排序数组合并为一个大的排序数组

具体步骤：

  1. 计算数组中点mid，将数组划分为左右两个子数组
  2. 重复上述操作直到不可再划分
  3. 依次回溯合并左右两个子数组

归并排序的递归写法最为常见，也是最好理解

但是因为递归需要占用\(O(logN)\)的栈帧内存，所以我们不太建议使用递归写法，还是更希望掌握迭代的写法

算法性能分析

  • 时间复杂度：\(O(NlogN)\)，很明显是\(logN\)层，然后每层\(N\)的操作，所以总的时间复杂度为\(O(NlogN)\)
  • 空间复杂度：\(O(N)\)，需要借助辅助数组，\(O(N)\)的空间开销，另外如果是递归，还需要\(O(logN)\)的栈帧空间，所以肯定是非原地排序
  • 稳定排序：在合并的时候可以保证顺序不变
  • 非自适应：任意数据时间复杂度都相同

算法的应用

链表排序

LeetCode 148

给你链表的头结点 head ，请将其按 升序 排列并返回 排序后的链表 。

请使用时间复杂度为\(Nlog(N)\)以及常数空间复杂度的算法

因为题目中要求使用时间复杂度为\(NlogN\)的时间复杂度，所以我们可以使用快排，堆排序以及归并排序，但是因为是链表形式，每次查询都是\(O(N)\)的时间复杂度，所以使用快排复杂度就不是之前的\(Nlog(N)\)了，而归并排序非常适用，并且不需要额外的数组空间

此外，由于题目要求常数空间复杂度，所以也不能使用堆排序，也不能使用归并排序的递归方式，我们直接使用迭代实现归并排序，因为递归需要\(log(N)\)的栈帧空间

链表的归并排序和普通的差不多，我们需要做的也是依次排序小链表，合并小链表即可，需要注意的就是要将每个小链表的尾部指向null即与其他的小链表分隔开来

合并排序链表

LeetCode 23

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

这道题第一想法就是用k个指针指向每个链表的头结点，然后每次比较取最小的头结点然后向后移动一位，但是这样的时间复杂度是\(log(KN)\)，还是指数级的

我们可以使用小顶堆存储每个头节点，然后每次都取出堆顶元素即可，维护堆的时间复杂度为\(logk\)，取堆顶元素的时间复杂度为\(O(1)\)，所以总的时间复杂度也就是\(O(Nlog(N))\)

我们可以看到，这道题是将几个排好序的合并起来，这其实就是归并排序，不过归并排序是两个子数组合并，那这里就是可以两两一组合并就行

快速排序

算法介绍

快速排序的基础思想是分治法的思想，核心思路就是找到一个基准值，然后将小于该基准值的元素放到他左边，大于该基准值的元素放到他右边，然后继续对左边和右边分别进行快速排序

  1. 我们以数组某一个元素为基准值，设为pivot
  2. 设置左右指针i,j
  3. 左指针向右移动，直到找到一个比pivot大的元素
  4. 右指针向左移动，直到找到一个比pivot小的元素
  5. 交换左右指针的值
  6. 重复上述操作，直到左右指针相遇
  7. 这个时候将pivot与指针所在值交换
  8. 这样pivot左边的值全部小于等于pivot，右边的值全部大于等于pivot

算法性能分析

  • 时间复杂度：\(O(NlogN)\)，很明显是\(logN\)层，然后每层\(N\)的操作，所以总的时间复杂度为\(O(NlogN)\)
  • 空间复杂度：\(O(N)\)，输入如果全部倒序的话需要最差递归空间\(O(N)\)，但是是原地排序
  • 非稳定排序：在划分交换的时候可能改变相同元素的相对顺序
  • 自适应：最差情况，即完全倒序情况，时间复杂度为\(O(N^2)\)

我们可以发现，基准值的选取对于时间复杂度的影响很大，例如我们选取最左边元素为基准值，那么当遇到倒序数据的时候时间复杂度最差，所以我们可以对基准值进行优化

比较常用的基准值我们可以选取首元素，中间元素，尾元素的中位数，这样基准值必然不会是最值

算法的应用

数组中第k大的数字

LeetCode 215

给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。

你必须设计并实现时间复杂度为 O(n) 的算法解决此问题。

这道题似曾相识，我们可以维护一个大小为k的大顶堆，但是这样时间复杂度就是\(O(Nlogk)\)了，题目中要求是时间复杂度为\(O(N)\)

这道题要求第k个最大的元素，我们发现快排中的partition操作其实就是将数组划分为了大于一个数和小于一个数，那么我们只需要得到基准值的下标为n-k即找到了第k个最大的值

堆排序

算法介绍

堆排序其实非常简单，就是使用大顶堆小顶堆的数据结构维护待排序数据，然后每次取出堆顶元素，这样连续取出的元素就是有序的，操作非常简单，关于堆的一些知识可以看堆那篇文章

算法性能分析

  • 时间复杂度：\(O(NlogN)\)，堆的维护花费\(O(log(N))\)，每个元素入堆出堆，所用总共时间复杂度为\(O(Nlog(N))\)
  • 空间复杂度：\(O(1)\)，这里的\(O(1)\)是指直接在数组上构建堆，而不是直接使用现成的堆，所以是就地排序
  • 非稳定排序：在划分交换的时候可能改变相同元素的相对顺序
  • 非自适应：与数据无关，时间复杂度恒定

计数排序

算法介绍

计数排序是一种线性时间的整数排序算法，如果数组长度为n，整数范围（最大值与最小值的差值）为k，对于k远小于n的情况下，计数排序的性能要显著优于其他几种排序

计数排序的思路如下：

  1. 找出数组的最小值和最大值，构造一个长度为max-min+1的数组
  2. 然后遍历数组，对于每个元素num，在计数数组中加一
  3. 最后遍历计数数组，依次输出计数个数的数

例如整数数组[2,3,4,2,3,2,1]，最大值为4，最小值为1

那么我们构建计数数组[0,0,0,0]

遍历数组，计数，得到计数数组为[1,3,2,1]，标志着1出现了1次，2出现了3次，3出现了1次，4出现了依次

最后根据计数数组输出[1,2,2,2,3,3,4]

算法性能分析

  • 时间复杂度：如果数组长度为n，整数范围为k，那么时间复杂度为\(O(n+k)\)，是十分高效的操作
  • 空间复杂度：\(O(k)\)，使用了辅助计数数组，长度为k
  • 非自适应：与数据无关，时间复杂度恒定

算法的应用

数组相对排序

LeetCode 1122

给你两个数组，arr1 和 arr2，arr2 中的元素各不相同，arr2 中的每个元素都出现在 arr1 中。

对 arr1 中的元素进行排序，使 arr1 中项的相对顺序和 arr2 中的相对顺序相同。未在 arr2 中出现过的元素需要按照升序放在 arr1 的末尾。

这道题因为是整数的有序输出，所以我们可以使用计数排序

对arr1中的所有数进行计数，然后先按arr2中顺序输出计数数组中的数并--，最后输出计数数组中非0的数字即可
