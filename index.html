Vector 和 ArrayList 区别

相同点：

  1. 继承了相同的父类实现了相同的接口
  2. 底层都是数组实现的
  3. 初始默认长度都是10

不同点：

  1. Vector中public方法多数加了synchronized，线程安全，ArrayList没有
  2. 扩容方式不同，ArrayList扩展后数组大小为原数组的1.5倍，Vector中多了扩展因子，当扩展因子大于0的时候，扩展后为原数组大小加上扩展因子大小，否则为原数组的两倍。

栈

栈的基础知识

栈是一种非常常见的数据结构，他的核心特点就是LIFO，后进先出。

对于栈，我们主要需要知道的是栈的三个主要函数（下面的栈我们使用ArrayList实现，所以对于容量为自动调整）

  • PUSH：即入栈，我们在栈顶放入元素，栈顶标识符++
  • POP：即出栈，我们取出栈顶元素，栈顶标识符--
  • PEEK：获取栈顶元素但不出栈，栈顶标识符不变

栈的应用

栈的应用其实主要就是后入先出的核心思想，当对于数据的存储顺序与使用顺序相反的时候即最后保存的数据最先使用，我们可以将这些数据存储到栈中

后缀（逆波兰）表达式

后缀表达式是一种算术表达式，他的操作符在操作数的后面，我们平时用的是中序表达式，即操作符在操作数中间。

例如下面式子： \[ 4.99+5.99+6.99*1.06 \] 我们自己肯定知道乘法的等级要高于加法，所以先算\(6.99*1.06\)再加上前面的和，但是对于计算器或者计算机，没有括号的辅助，很有可能就是直接按照顺序进行计算，就会造成错误。但是我们来看后缀表达式 \[ 4.99\space5.99\space+\space6.99\space1.06\space*\space+ \] 上面的表达式表示我们先\(4.99+5.99\)计为\(A_1\)，然后\(6.99*1.6\)计为\(A_2\)，最后再将\(A_1,A_2\)相加

使用后缀表达式，就已经表明了计算的顺序，无须借用括号来表明。

计算后缀表达式的方法最容易的就是使用栈来操作：当遇到一个数，那么我们就把他入栈，遇到表达式弹出栈顶两个数进行计算，将结果入栈。

计算后缀表达式的值

LeetCode 150

给你一个字符串数组 tokens ，表示一个根据 逆波兰表示法 表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

输入：tokens = ["2","1","+","3","*"]

输出：9

解释：该算式转化为常见的中缀算术表达式为：\(((2 + 1) * 3) = 9\)

中缀转后缀表达式

输入：tokens = ["a","+","b","*","c","+","(","d","*","e","+","f",")","*","g"]

输出：abc*+de*f+g*+

题目中中序表达式为 \[ a+b*c+(d*e+f)*g \] 我们需要转换为 \[ abc*+de*f+g*+ \] 中序转后缀表达式的策略就是：

  • 操作数：直接输出
  • 操作符：对于操作符我们使用一个栈来存储操作符。当遇到左括号时，直接入栈，遇到右括号时，弹出栈顶所有操作符并输出直到遇到左括号，弹出左括号，左括号不输出。当遇到其他操作符时，弹出栈顶操作符并输出直到栈顶操作符优先级低于改操作符，但是遇到左括号停止。其中+-优先级较低，*/优先级较高

小行星碰撞

LeetCode 735

给定一个整数数组 asteroids，表示在同一行的小行星。

对于数组中的每一个元素，其绝对值表示小行星的大小，正负表示小行星的移动方向（正表示向右移动，负表示向左移动）。每一颗小行星以相同的速度移动。

找出碰撞后剩下的所有小行星。碰撞规则：两个行星相互碰撞，较小的行星会爆炸。如果两颗行星大小相同，则两颗行星都会爆炸。两颗移动方向相同的行星，永远不会发生碰撞。

输入：asteroids = [4,5,-6,4,8,-5]

输出：[-6,4,8]

这道题乍一看比较新颖，但其实观察碰撞规律之后就可以发现非常简单。

第一个行星4，不知道其他行星，先放到容器，看第二个行星5，跟第一个同向，所以同样也放到容器，第三个行星-6，方向与前面两个不同，所以先看上一个行星5，所以碰撞之后剩下-6，然后再看第一个行星4，也会碰撞，所以最后只剩下-6，依次类推。可以很明显发现符合栈的后入先出的特点。

要注意的其实就是几个条件的判断，理清逻辑就很简单了。

表：数组和链表

对于表，我们可以普遍地认为有数组和链表两种实现形式，即能够连续存储一些数的抽象数据类型（ADT）

表的数组实现

基础知识

数组是一种非常简单的数据结构，是由相同类型的元素组成的数据集合，在内存中按顺序连续存储数据。

对于数组的数据，我们可以非常容易的通过下标获取和修改数据，其时间复杂度为\(O(1)\)，但是对于插入数据和删除数据则非常的复杂，因为插入删除数据涉及到需要将部分数据全部后移或者迁移一位，开销非常大。

此外，创建数组的时候需要指定数组的大小，并且根据容量分配内存，这就导致有的时候我们为了初始化较大容量的数组结果空间利用效率极低。为了处理空间效率不高的情况，便设计出了动态数组，会自动根据数组内部容量分配新的空间。

Springboot整合springsession实现session共享

简介

session我们之前有介绍过（可见springboot整合springsecurity），简单来说就是将用户信息或者数据存储在服务器上，通常用于验证用户身份或者避免通过获取相关信息。

但是，缺点也是非常明显：

  1. 占用服务器负载：我们可以使用token，用时间换取空间

  2. 对于多服务器环境，session无法共享

对于第二点缺点，我们目前有几种比较常用的解决方法

  1. 使用cookie加密的方式将session保存在客户端上

    优点是可以减轻服务器压力，缺点是每次请求都要带上cookie信息，占用一定带宽。此外若用户禁用cookie则无法使用

  2. 服务器间同步

    通过配置tomcat集群，在集群中广播自己的session信息，但是缺点很明显，当集群规模较大时，会占用大量资源来进行session同步处理

  3. 基于分布式缓存的session共享机制

    将session缓存到redis中，这样不同服务器都可以直接到内存中获取session，效率高，也最常用

SpringBoot整合Redis实现缓存

Redis

redis简单来说就是一款基于内存的以键值对方式存储数据的非关系型数据库。

优点非常明显，由于存储在内存中，所以读取非常快。这种速度是mysql等数据库完全不能比拟的，所以我们常用redis作为缓存以减少对本地数据库的查询。

redis可以作为系统进程间的资源共享，例如使用redis实现spring session共享等等

Springboot整合RabbitMQ

MQ

简介

消息队列（Message Queue，简称MQ），只保存消息的一个容器，本质上是一个容器。

简单来说，消息队列就是一个存储消息的容器，它接受消息发布者发布的消息，然后将消息存储在容器中等待之后的消费者到容器里消费消息。

这就是最简单的一个消息队列的模型

Producer：消息生产者，负责产生和发布消息到Broker中

Broker：消息处理中心，负责消息的存储，确认等等，一般其中会包含多个queue

Consumer：消息消费者，负责从Broker中获取消息，并进行相关处理（消费）

Springboot实现异步调用

概述

我们之前的开发中往往都是直接按照逻辑写下去，不用考虑系统时间开销。这就是同步调用，即我的下一步操作必须等待上一步操作完成，但是，在一些场景下，我们希望使用异步调用而不是同步调用，即我让另外的资源去执行这个操作，而我的主逻辑不用等待这个操作的完成，这样毫无疑问节省了大量的时间。

  • 同步调用：程序按照逻辑顺序依次执行，每一个操作都必须等待上一个操作完成后才能执行
  • 异步调用：程序在执行过程中不必等待异步调用的完成即可完成之后的操作

很经典的例子就是去银行办理业务：

我今天计划要去银行办理一个业务，并且还打算去买个衣服，那么我先去银行，发现需要排队等待（即某个逻辑执行需要一段时间），那我取个号（开辟新的线程去执行），然后就直接去买衣服（继续执行主逻辑）。买完衣服后我再回来等待。
